## Environment setup for app-auth login

Visual Note uses the app-owned auth system backed by Supabase Postgres tables. Set these environment variables before logging in:

- `NEXT_PUBLIC_SUPABASE_URL`
- `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`
- `SUPABASE_SERVICE_ROLE_KEY`

Create or update `.env.local` with all three values, then restart `npm run dev`.

To get the `SUPABASE_SERVICE_ROLE_KEY`, open Supabase → your project → Project Settings → API and copy the `service_role` secret key.  
Do not commit this key to version control.

If this key is missing, `/api/auth/session` returns `authReady: false`, login/register buttons are disabled, and the app shows a configuration notice instead.
